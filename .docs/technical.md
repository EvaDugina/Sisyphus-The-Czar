## Паспорт

- **Проект:** Sisyphus The Czar
- **Стадия:** POC B
- **Последнее обновление:** 2026-07-14

---

## Архитектура

- **Компоненты:** браузерный клиент, HTTP/WebSocket-сервер, менеджер сессий, общее ядро физики.
- **Потоки данных:** при загрузке без `session` клиент автоматически создаёт комнату через HTTP и заменяет URL; при наличии `session` сразу подключается к ней по WebSocket. Затем отправляет управление и каноническое положение указателя не чаще 30 раз в секунду; сервер рассчитывает физику, рассылает снимки 20 раз в секунду и ретранслирует курсоры.
- **Внешние зависимости:** nginx хоста завершает TLS и проксирует HTTP/WebSocket. БД, очередь и внешние API не используются.

Сервер считает физику фиксированным шагом 60 Гц в канонической сцене `1000×5000`, соответствующей странице высотой `500vh`. Браузеры преобразуют координаты в локальную геометрию и рисуют между снимками через 90-миллисекундную интерполяцию. Управляющий браузер вычисляет максимальный `y` верхнего края с учётом адаптивной высоты камня и границы `80vh`; сервер валидирует его в пределах канонических `0…800` и непрерывно останавливает камень на drag- или physics-шаге.

## Модули и ответственность

- `server` — HTTP, WebSocket Upgrade, Origin/rate-limit, heartbeat и жизненный цикл процесса.
- `SessionManager` — комнаты, участники, блокировка управления, таймеры, TTL и рассылка снимков.
- `shared/physics` — чистые расчёты падения, броска, отскока и нормализация параметров.
- Браузерный клиент — pointer-ввод, локальный отзыв управляющему, reconnect, интерполяция камня и сглаженная отрисовка удалённых рук.

## Данные и контракты

### Сущности

- **Сессия:** случайный 128-битный `id`, состояние камня, общая физика, клиенты, `revision`, страховочный TTL и до 1000 точек траектории. После явного выхода последнего клиента удаляется немедленно.
- **Локальный след:** браузер по умолчанию хранит заданное число точек; личная настройка «Без ограничения длины» отключает локальную обрезку. Серверный буфер остаётся ограничен 1000 точками, поэтому новый участник получает только последнюю часть истории.
- **Состояние камня:** `phase`, `x`, `y`, `vx`, `vy`, `dragging`, `controllerId`, `turbTime`.
- **Клиент:** случайный `clientId` из `sessionStorage`, приватный одноразовый `leaveToken`, последнее `seq`, WebSocket, момент отключения и последний указатель `{x,y,mode,visible}`.

### API

- `POST /api/sessions` → принимает нормализованные `state`, `physics`, `trail`; возвращает `sessionId`, `expiresAt`.
- `POST /api/sessions/:id/leave` → принимает `clientId` и персональный `leaveToken`; удаляет клиента, а при пустой комнате сразу удаляет сессию. Повторный запрос к уже удалённой комнате идемпотентен.
- `GET /healthz` → возвращает статус процесса и число комнат.
- `WS /realtime?session=<id>&client=<id>` → оболочка `{v,type,seq,payload}`; поддерживает `control.acquire/move/release`, `pointer.update`, `physics.update`, `ping`. Управляющие сообщения передают валидируемый `stopMaxY` для адаптивной верхней границы; серверные `pointer.update` и `presence.update` дают другим браузерам текущие `grab/grabbing`-курсоры.

HTTP-тело ограничено 16 КиБ, WebSocket-сообщение — 64 КиБ. Сервер проверяет диапазоны, версию протокола, тип и монотонный `seq`.
Масса валидируется в диапазоне `1–100`; значения выше 100 сервер обрезает до верхней границы.

## Окружение и конфигурация

| Переменная | Назначение | Пример |
|---|---|---|
| `DEBUG` | dev-режим и послабление same-origin для локальной разработки | `false` |
| `ALLOWED_ORIGIN` | разрешённые публичные origin через запятую | `https://example.com` |
| `SESSION_TTL_SECONDS` | TTL после последней активности | `86400` |
| `PORT` | внутренний порт контейнера | `8080` |

- `DEBUG=true` — polling-наблюдатель перезапускает Node при изменениях `server/` и `shared/`; изменения `index.html` и `assets/` обновляют браузер без рестарта сервера. Dev-кэш отключён, Origin разрешён для локальной разработки.
- `DEBUG=false` — same-origin/allowlist, CSP, CORP, production-кэш ассетов и без автообновления.

## Развёртывание

- **Локальный запуск:** `docker compose -f docker-compose.dev.yml up --build`; после первого запуска исходники подхватываются из bind mounts без пересборки контейнера.
- **Деплой:** `bash deploy.sh` идемпотентно собирает production-образ, обновляет один контейнер и показывает его статус.
- Приложение публикуется только на `127.0.0.1:18082`; HTTPS/WSS и WebSocket Upgrade настраиваются в nginx хоста.

Требования для двух пользователей: Ubuntu Server 24.04 LTS, 1 vCPU, минимум 512 МБ RAM, 2 ГБ диска, канал от 1 Мбит/с. Рекомендуются 1 ГБ RAM, 10 Мбит/с, RTT до 100 мс, jitter до 30 мс. Запускается один экземпляр: горизонтальное масштабирование без общего хранилища запрещено.

## Безопасность

- Секреты и production-конфигурация только в `.env`; `.env` исключён из git и Docker build context.
- Session ID генерируется из 128 криптографически случайных бит.
- Проверяются Origin, размеры сообщений, типы и диапазоны; создание комнат и подключения ограничены по IP.
- Закрытие вкладки отправляет `sendBeacon`; приватный `leaveToken` не позволяет участнику удалить чужое подключение по видимому `clientId`.
- CSP запрещает фреймы, объекты, камеру, микрофон и геолокацию. `Referrer-Policy: no-referrer` не даёт ссылке комнаты уйти к внешнему шрифту.
- Контейнер запускается не от root, read-only, с `no-new-privileges`, лимитами CPU/RAM/PID и loopback-портом.

## Ограничения этапа

- Нет БД, восстановления комнат после рестарта, горизонтального масштабирования и бэкапов.
- Нет аккаунтов, ролей, MFA, аудита действий, продуктовой аналитики и полноценного мониторинга.
- Нет гарантии качества при RTT выше 100 мс или нестабильной мобильной сети.
- Аварийное завершение браузера может не доставить `pagehide` beacon; такие комнаты остаются до страховочного TTL.
- TLS-сертификат и nginx — инфраструктура хоста, не часть репозитория.

## Технические решения

- **2026-07-14:** клиентская физика расходилась бы между экранами → сервер стал авторитетным, координаты переведены в каноническую сцену → требуется постоянный WebSocket.
- **2026-07-14:** отдельные nginx и realtime-контейнеры увеличивали операционную сложность → один Node-процесс отдаёт статику, API и WebSocket → один порт и один deploy.
- **2026-07-14:** Redis/PostgreSQL избыточны для POC B → комнаты хранятся в памяти 24 часа → один экземпляр и потеря данных при рестарте приняты явно.

## Frontend

- **Стек:** HTML/CSS/JavaScript без сборщика.
- **Безопасность:** секретов и приватных API в браузере нет; запросы идут только на свой origin.
- **Состояния:** автоматическое создание, подключение, online, занятое управление, 400-мс подтверждение копирования, удалённые `grab/grabbing`-курсоры, reconnect, истёкшая сессия, автоматическая остановка в верхней зоне и ограниченный/неограниченный локальный след. Локальный режим остаётся fallback для `file:` и ошибки создания.
- **Адаптивность:** desktop и mobile используют локальную геометрию поверх канонических координат.
