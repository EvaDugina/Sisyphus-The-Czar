## Функциональные требования

- **Общая блокировка прокрутки:** Пока в комнате не было первого принятого касания камня, у всех участников должна быть скрыта вертикальная полоса прокрутки и заблокирована прокрутка колесом, клавиатурой и touch-жестами. После первого принятого `pointerdown` прокрутка разблокируется одновременно у всех участников.
  - **Пояснение:** До начала общего сценария пользователь должен оставаться на первой экранной странице; первое касание переводит всю комнату в прокручиваемое состояние.
  - **Источник:** "До первого клика по камню не отображай overflow-y и запрещай scroll"; "у всех участников комнаты сразу".
  - **Acceptance hint:** До касания `scrollY` остаётся нулевым в двух браузерах; после принятого касания оба браузера могут прокручивать страницу.

- **Видимый след:** При включённой настройке след должен быть заметен с начала движения камня и отображаться по всей сцене высотой `10000vh`.
  - **Пояснение:** След должен оставаться рабочим независимо от большой высоты сцены, прокрутки, resize и текущей темы.
  - **Источник:** "не рисуется / не виден след".
  - **Acceptance hint:** После начала падения появляется различимая линия или точки; след остаётся видимым при прокрутке, resize и смене темы.

- **Светлая начальная тема:** Новая, ещё не затронутая сессия должна отображаться в светлой теме, включая загрузку после перехода с `/` и получение первого server snapshot. С началом падения интерфейс переключается на тёмную тему.
  - **Пояснение:** Начальный экран должен выглядеть одинаково до первого взаимодействия, а существующий переход к тёмной теме после начала падения сохраняется.
  - **Источник:** "Изначально тема должна быть светлой"; "да, светлая тема нужна только до первого касания".
  - **Acceptance hint:** До первого касания новая комната светлая; после запуска падения становится тёмной.

- **Стабильный масштаб камня:** После отделения от руки камень должен возвращаться к масштабу `1`, без дополнительного уменьшения. Увеличение непосредственно во время удержания сохраняется.
  - **Пояснение:** Смена состояния `dragging → falling` не должна восприниматься как сжатие камня.
  - **Источник:** "После того как камень отклеивается от руки он как-то странно слегка уменьшается - убери это".
  - **Acceptance hint:** Визуальные размеры камня до захвата и во время свободного падения совпадают.

- **Диапазон тяготения:** Регулятор и серверная валидация должны поддерживать значения от `1` до `10`, шаг `0.1`, значение по умолчанию `1`.
  - **Пояснение:** Максимальное значение должно действительно участвовать в серверной физике и быть общим для комнаты.
  - **Источник:** "Дай возможность увеличить тяготение в пределах от 1 до 10".
  - **Acceptance hint:** Значение `10` сохраняется в UI, принимается сервером, влияет на ускорение и синхронизируется другому участнику.

- **Надёжное применение параметров:** Все настройки должны применяться к соответствующему поведению. Общая физика отправляется серверу и синхронизируется всем участникам; изменения до готовности WebSocket отправляются после подключения; локальные параметры следа применяются и перерисовываются сразу; server snapshot обновляет UI без циклической повторной отправки.
  - **Пояснение:** UI не должен показывать значение, которое фактически не используется клиентом или авторитетной серверной моделью.
  - **Источник:** "Измененные параметры из UI с параметрами не применяеются как будто!".
  - **Acceptance hint:** Каждый общий физический параметр проверяется через server snapshot и второй браузер; параметры следа визуально применяются в текущем браузере.

- **Удаление кнопки из настроек:** Кнопка «Поделиться сессией» / «Скопировать ссылку» / «Создать новую сессию» должна быть полностью удалена из панели параметров. Статус комнаты остаётся, а копирование URL доступно только через верхнюю кнопку рядом с настройками.
  - **Пояснение:** В панели не должно быть дублирующего или меняющего назначение действия сессии.
  - **Источник:** "Убери скопировать ссылку из ui параметров"; "удалить из панели всю кнопку целиком".
  - **Acceptance hint:** В открытой панели параметров нет кнопки сессии; верхняя кнопка продолжает копировать текущий URL.

- **Сохранение состояния при reload:** Обновление страницы по `/?session=<id>` не должно удалять или создавать заново комнату. После reconnect сохраняются ID комнаты, фаза, положение и скорость камня, отпечаток, серверная история следа и общие параметры физики. Движущийся камень может продолжать серверную физику во время reload, но не должен возвращаться в исходное состояние.
  - **Пояснение:** Reload является кратким переподключением участника, а не завершением комнаты.
  - **Источник:** "При обновлении страницы не обнуляй состояние сессии!".
  - **Acceptance hint:** До и после reload используется один session ID; состояние не сбрасывается к параметрам новой комнаты.

- **Продление активной комнаты:** Пока в комнате есть подключённый или быстро переподключающийся участник, сервер должен продлевать `expiresAt` и не допускать удаления комнаты по TTL.
  - **Пояснение:** Активная демонстрация не должна завершиться из-за таймера хранения.
  - **Источник:** "При истекании сессии - продлевай её".
  - **Acceptance hint:** Подключение и активность сдвигают `expiresAt`; активная комната не удаляется на старой границе TTL.

- **Удаление пустой комнаты с задержкой:** После закрытия последней вкладки комната должна перейти в короткий grace-период. Возврат участника через reload/reconnect сохраняет комнату и её состояние; если никто не вернулся, комната удаляется из памяти.
  - **Пояснение:** Grace-период отличает краткое обновление страницы от окончательного ухода последнего участника.
  - **Источник:** "Удалить после короткого grace-периода, достаточного для reload/reconnect".
  - **Acceptance hint:** Обычный reload сохраняет комнату; закрытая последняя вкладка приводит к удалению после настроенного grace-периода.

## Нефункциональные требования

- **Устойчивый рендер следа:** След должен работать на desktop и mobile без canvas, превышающего браузерные ограничения из-за высоты сцены `10000vh`.
  - **Пояснение:** Большая высота документа не должна приводить к пустому canvas или отказу рисования.
  - **Источник:** "не рисуется / не виден след".
  - **Acceptance hint:** След проверяется в поддерживаемых desktop- и mobile-размерах после длинного падения и прокрутки.

- **Доступность управления при scroll-lock:** Блокировка прокрутки не должна блокировать верхние кнопки, панель параметров и первый `pointerdown` по камню.
  - **Пояснение:** Ограничивается только прокрутка документа, а не интерактивность стартового экрана.
  - **Источник:** "До первого клика по камню ... запрещай scroll".
  - **Acceptance hint:** Настройки, копирование ссылки и захват камня работают до снятия scroll-lock.

- **Согласованность общей комнаты:** Reload и изменение общих параметров не должны запускать отдельную локальную физику или создавать расходящееся состояние между участниками.
  - **Пояснение:** Сервер остаётся авторитетным источником состояния и физики.
  - **Источник:** "При обновлении страницы не обнуляй состояние сессии"; "параметры ... не применяются".
  - **Acceptance hint:** Два браузера после reload и изменения параметров получают одинаковые server snapshots.

- **Ограниченный жизненный цикл пустых комнат:** Grace-период не должен приводить к бессрочному накоплению полностью пустых комнат в памяти.
  - **Пояснение:** Сохранение reload не отменяет очистку окончательно покинутых сессий.
  - **Источник:** "Удалить после короткого grace-периода".
  - **Acceptance hint:** Пустая комната удаляется после grace-периода и исчезает из счётчика `/healthz`.

## Проблемы и замечания, превращённые в требования

- **След не отображается:** Сейчас backing canvas рассчитывается по высоте всей сцены и может превышать ограничения браузера. Требуемое поведение: след стабильно отображается по всей траектории на сцене `10000vh`.
  - **Источник:** "не рисуется / не виден след".
  - **Acceptance hint:** После движения камня canvas содержит видимые пиксели в текущем viewport.

- **Камень уменьшается после отпускания:** Сейчас состояние свободного падения задаёт масштаб меньше обычного. Требуемое поведение: после отпускания используется масштаб `1`.
  - **Источник:** "слегка уменьшается - убери это".
  - **Acceptance hint:** Bounding box камня до захвата и после отпускания совпадает с допустимой погрешностью рендера.

- **Тяготение ограничено значением `2`:** Сейчас UI и общая валидация не принимают требуемое значение `10`. Требуемое поведение: клиент и сервер используют диапазон `1–10`.
  - **Источник:** "увеличить тяготение в пределах от 1 до 10".
  - **Acceptance hint:** Unit-тесты принимают границы `1` и `10`, а значения вне диапазона нормализуются к ближайшей границе.

- **Изменения параметров могут теряться:** Сейчас обновление до готовности WebSocket не ставится в очередь, а server snapshot может вернуть UI прежние значения. Требуемое поведение: последнее пользовательское изменение гарантированно применяется после подключения.
  - **Источник:** "параметры ... не применяеются как будто".
  - **Acceptance hint:** Параметр, изменённый во время connecting, после перехода online присутствует в server snapshot.

- **Reload удаляет последнюю комнату:** Сейчас `pagehide` последнего участника может немедленно уничтожить сессию. Требуемое поведение: reload проходит через reconnect grace и сохраняет серверное состояние.
  - **Источник:** "При обновлении страницы не обнуляй состояние сессии".
  - **Acceptance hint:** Reload единственного участника не приводит к `session_not_found` и не создаёт новую комнату.

- **Копирование продублировано в панели:** Сейчас панель параметров содержит дополнительную кнопку сессии. Требуемое поведение: действие копирования остаётся только в верхней кнопке.
  - **Источник:** "Убери скопировать ссылку из ui параметров".
  - **Acceptance hint:** В панели отсутствует соответствующая кнопка и её обработчик.

## Не включено / требует уточнения

- Восстановление комнат после рестарта серверного процесса или контейнера не входит в требования.
- Постоянная база данных для хранения сессий не требуется.
- Увеличенный масштаб камня во время удержания не изменяется.
- Кнопка создания или восстановления комнаты внутри панели параметров не предусматривается.
- Бессрочное хранение полностью пустых комнат не требуется.
