# Fix 2026-07-22 12:30

## Контекст

Пользователь сообщил, что после настройки `50vh` полупрозрачный камень/след для активации светлой темы и дождя визуально не находится по центру верхней страницы. На скриншоте цель смещена вниз: `50vh` выглядит как позиция верхнего края объекта, а не его центра.

## Функциональные требования

1. Полупрозрачный камень/след должен располагаться по центру верхней страницы при открытии сцены сверху.
2. Значение `50vh` должно трактоваться как координата визуального центра камня/следа, а не как координата его верхнего края.
3. Горизонтальный центр камня/следа должен совпадать с горизонтальным центром viewport.
4. Вертикальный центр камня/следа должен совпадать с вертикальным центром первого viewport.
5. Зона попадания камня должна соответствовать отображаемому положению полупрозрачного следа.

## Нефункциональные требования

1. Позиционирование должно учитывать фактический размер камня и корректно работать при разных размерах viewport.
2. Масштабирование камня не должно смещать его визуальный центр относительно рассчитанной цели.
3. Существующая логика активации светлой темы и дождя не должна меняться, кроме корректировки позиции цели.

## Критерии приемки

1. После запуска сцены центр `.rock-imprint` находится примерно в точке `window.innerWidth / 2`, `window.innerHeight / 2`.
2. Визуальная цель не выглядит смещенной вниз на половину высоты камня.
3. Smoke-проверка подтверждает центрирование цели в верхнем viewport.
