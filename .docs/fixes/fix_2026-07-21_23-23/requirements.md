## Функциональные требования

- **Доступность gachi-модуля:** Backend в dev-окружении должен отдавать `/shared/gachi-sounds.js` с HTTP 200 и JavaScript-содержимым.
  - **Пояснение:** Клиент импортирует этот shared-модуль до запуска приложения; ошибка загрузки блокирует инициализацию страницы.
  - **Источник:** «Сайт не открывается» и скриншот с `404 (Not Found)` для `/shared/gachi-sounds.js`.
  - **Acceptance hint:** Прямой GET-запрос к `/shared/gachi-sounds.js` возвращает 200 и содержимое модуля.

- **Рабочий запуск приложения:** После исправления корневая страница dev-сервера должна загружаться и отображать игровую сцену без блокирующей ошибки импорта gachi-модуля.
  - **Пояснение:** Исправление должно восстановить пользовательский сценарий открытия сайта.
  - **Источник:** «Сайт не открывается».
  - **Acceptance hint:** Открытие `http://127.0.0.1:18082` приводит к работающей игровой странице.

  - **Пояснение:** Локальная разработка не должна ломаться из-за различий в способе доставки модулей.
  - **Источник:** 404 возникает при открытии dev-сайта после добавления нового shared-модуля.
  - **Acceptance hint:** Маршрут покрыт автоматическим регрессионным тестом.

- **Сохранение поведения:** Исправление не должно менять назначение и воспроизведение gachi-звуков или другие игровые механики.
  - **Пояснение:** Требуется исправить только доставку уже реализованного модуля.
  - **Источник:** Утверждённый объём исправления.
  - **Acceptance hint:** Изменения ограничены маршрутом публикации и его тестом.
