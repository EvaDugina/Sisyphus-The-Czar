## Функциональные требования

- **Размер руки:** Система должна отображать руку в два раза меньше текущего размера.
  - **Пояснение:** Масштаб визуального элемента руки уменьшается до 50% от текущего.
  - **Источник:** "Уменьши руку в 2 раза"
  - **Acceptance hint:** При одинаковом viewport габариты руки по ширине и высоте составляют примерно половину прежних.

- **Звук кандалов по клику:** Для пользователей, не относящихся к роли `slave`, система должна воспроизводить звук кандалов как при hover камня, так и при клике на камень.
  - **Пояснение:** Существующий hover-сценарий сохраняется и дополняется click-сценарием.
  - **Источник:** "Воспроизводи звук кандалов не только по hover, но и по клику на камень"
  - **Acceptance hint:** Наведение и отдельный клик по камню запускают звук кандалов.

- **Gachi-звук для slave:** Для пользователя с ролью `slave` система должна воспроизводить назначенный ему звук из папки `gachi` только по клику на камень.
  - **Пояснение:** Hover по камню у `slave` не должен запускать никакой звук.
  - **Источник:** "Для slaves воспроизводи звук из папки gachi... У slave по hover ничего не воспроизводится"
  - **Acceptance hint:** У `slave` hover бесшумен, а клик по камню запускает назначенный gachi-звук.

- **Сессионная привязка звука:** При первом появлении `slave` в текущем запуске сервера система должна назначить ему один конкретный звук из папки `gachi` и использовать этот звук для всех последующих кликов этого пользователя до завершения запуска сервера.
  - **Пояснение:** Привязка хранится только в оперативном состоянии сервера; постоянное хранилище не требуется.
  - **Источник:** "Каждому slave при первом попадании на сервер приписывай в соответствие один конкретный звук"; уточнение: "Пока жива сессия"
  - **Acceptance hint:** Повторные входы одного `slave` до перезапуска сервера сохраняют звук; после перезапуска допустимо новое назначение.

- **Целевая область для камня:** Наверху страницы система должна отображать полупрозрачный след или контур камня, обозначающий место, куда нужно поместить камень.
  - **Пояснение:** Элемент служит визуальной целью для игрового действия.
  - **Источник:** "Наверху страницы отображай полупрозрачный след от камня"
  - **Acceptance hint:** В верхней части игрового поля постоянно видна полупрозрачная область, соответствующая форме или силуэту камня.

- **Активация светлой темы и дождя:** Когда камень помещён в целевую область наверху страницы, система должна включить светлую тему и запустить дождь.
  - **Пояснение:** Оба эффекта запускаются одним успешным попаданием камня в цель.
  - **Источник:** "...в который нужно поместить камень чтобы наступила светлая тема и пошел дождь"
  - **Acceptance hint:** После попадания камня в цель оформление становится светлым и визуально начинается дождь.

- **Удаление начальной скорости:** Система не должна отображать в UI настройку начальной скорости.
  - **Пояснение:** Настройка не влияет на поведение и удаляется из пользовательского интерфейса.
  - **Источник:** "Начальная скорость сейчас не используется убери её из UI"
  - **Acceptance hint:** В интерфейсе отсутствует поле, ползунок или подпись начальной скорости; остальные настройки продолжают работать.

- **Видимость траектории:** Система должна отображать линию траектории камня с достаточным контрастом в тёмной теме.
  - **Пояснение:** Цвет и прозрачность линии должны позволять уверенно различать её на тёмном фоне.
  - **Источник:** "Сейчас на темной теме не видно линии траектории камня"
  - **Acceptance hint:** В тёмной теме линия визуально различима на всём участке траектории.

## Нефункциональные требования

- **Предсказуемость аудио:** Звуковое поведение должно однозначно зависеть от роли пользователя и типа взаимодействия.
  - **Пояснение:** Для `slave` исключается hover-звук и звук кандалов по клику; для остальных сохраняется звук кандалов по hover и клику.
  - **Источник:** Требования о разных звуках для slaves и остальных пользователей.
  - **Acceptance hint:** Ролевые сценарии не смешиваются при повторных наведениях, кликах и переподключениях.

- **Стабильность назначения в сессии:** Назначенный `slave` gachi-звук должен оставаться неизменным до перезапуска сервера.
  - **Пояснение:** Персистентное хранение между запусками сервера не требуется.
  - **Источник:** "Пока жива сессия"
  - **Acceptance hint:** В рамках одного процесса сервера пользователь всегда получает один и тот же файл.

- **Читаемость визуальных подсказок:** Целевая область и линия траектории должны быть заметны на соответствующем фоне, не перекрывая основные игровые элементы.
  - **Пояснение:** Подсказки должны помогать действию и не создавать визуального шума.
  - **Источник:** Требования о полупрозрачном следе и невидимой траектории.
  - **Acceptance hint:** Подсказки различимы, камень и рука остаются визуально приоритетными.

  - **Acceptance hint:** Линия различима при активной тёмной теме на поддерживаемых размерах экрана.
