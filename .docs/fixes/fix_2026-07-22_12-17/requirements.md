## Исходный текст

Пользователь просит сделать сохранение конфигураций лаконичнее:

- Использовать вариант 1.
- Кнопку сохранения сделать иконкой.
- Выбор версии переместить наверх.
- Если после загрузки шаблона UI-параметров параметры изменились до сохранения, переключаться на "Черновик".

## Функциональные требования

- **Компактный блок версий:** Система должна отображать выбор версии настроек наверху блока сохранения конфигураций.
  - **Пояснение:** Выбор сохраненной версии должен быть первым элементом блока.
  - **Источник:** "Выбор версии перемести наверх"
  - **Acceptance hint:** Первым элементом блока является select выбора версии.

- **Иконка сохранения:** Система должна заменить текстовую кнопку "Сохранить" на компактную кнопку с иконкой сохранения.
  - **Пояснение:** Действие сохранения должно занимать меньше места и оставаться доступным.
  - **Источник:** "Кнопку сохранить сделай иконкой"
  - **Acceptance hint:** Кнопка сохранения отображается как иконка и имеет доступное имя через `aria-label`, `title` или подсказку.

- **Однострочное сохранение версии:** Система должна сделать сохранение конфигурации лаконичным: поле имени версии и кнопка сохранения должны быть компактно сгруппированы.
  - **Пояснение:** Блок должен быть визуально ниже и проще текущей разметки.
  - **Источник:** "вариант 1"
  - **Acceptance hint:** Блок занимает меньше высоты, чем текущий вариант с отдельными строками "Название версии", "Сохранить", "Переименовать", "Версии".

- **Переход в черновик при изменениях:** Если пользователь загрузил сохраненную версию UI-параметров и затем изменил параметры до сохранения, система должна переключить текущий статус/выбор на "Черновик".
  - **Пояснение:** Пользователь должен видеть, что текущие параметры уже не совпадают с выбранной сохраненной версией.
  - **Источник:** "Если после загрузки шаблона UI параметров параметры измененились до сохранения переключайся на черновик"
  - **Acceptance hint:** После изменения любого параметра выбранная сохраненная версия больше не выглядит активной; вместо нее отображается "Черновик".

## Нефункциональные требования

- **Лаконичность:** Блок сохранения конфигураций должен занимать меньше высоты и визуально меньше шуметь в панели настроек.
  - **Пояснение:** Пользователь должен быстрее сканировать панель настроек.
  - **Источник:** "Сделай лаконичнее сохранения конфигураций"
  - **Acceptance hint:** Блок визуально компактнее текущего состояния.

- **Понятность:** Иконка сохранения должна быть понятна через `aria-label`, `title` или существующий механизм подсказок.
  - **Пояснение:** Иконка не должна ухудшить доступность действия сохранения.
  - **Источник:** "Кнопку сохранить сделай иконкой"
  - **Acceptance hint:** У кнопки есть доступное текстовое описание.

- **Сохранение поведения:** Выбор версии должен по-прежнему сразу применять сохраненные настройки.
  - **Пояснение:** Компоновка UI не должна ломать существующий сценарий применения версии.
  - **Источник:** "Выбор версии перемести наверх"
  - **Acceptance hint:** При выборе версии значения контролов панели применяются без дополнительного действия.

## Проблемы и замечания, превращённые в требования

- **Слишком громоздкий блок сохранения:** Сейчас сохранение конфигураций занимает несколько строк и выглядит тяжеловато. Требуемое поведение: сделать блок компактнее.
  - **Источник:** "Сделай лаконичнее сохранения конфигураций"
  - **Acceptance hint:** В блоке меньше постоянно видимых текстовых элементов и действий.

- **Нет явного состояния несохраненных изменений:** Сейчас после изменения параметров загруженная версия может визуально оставаться выбранной. Требуемое поведение: показывать "Черновик".
  - **Источник:** "Если после загрузки шаблона UI параметров параметры измененились до сохранения переключайся на черновик"
  - **Acceptance hint:** После изменения параметра select или статус версии показывает "Черновик".
