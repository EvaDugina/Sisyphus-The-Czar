# Sisyphus The Czar

Стадия: POC B

Интерактивная веб-миниатюра: пользователь запускает падение камня, затем поднимает его из нижней тёмной сцены к верхней цели.

## Локальный запуск

Откройте `index.html` в браузере или запустите статический сервер:

```bash
python3 -m http.server 8080
```

После запуска через Python страница будет доступна на `http://127.0.0.1:8080/`.

## Docker

```bash
docker compose up -d --build
```

Контейнер слушает только локальный интерфейс сервера: `127.0.0.1:18082`.

Prod-образ **запекает** `index.html` и `assets/` внутрь себя (`COPY` в `Dockerfile`),
поэтому после правок его нужно пересобирать (`up -d --build`).

## Dev-режим (live-reload без пересборки)

Для разработки есть отдельная конфигурация, где nginx отдаёт файлы напрямую с диска
через bind-mount — правки в `index.html` / `assets` видны после обновления страницы,
контейнер пересобирать и перезапускать не нужно:

```bash
docker compose -f docker-compose.dev.yml up -d
```

Открыть `http://127.0.0.1:18082/`, отредактировать `index.html`, обновить вкладку —
изменения на месте. Остановить: `docker compose -f docker-compose.dev.yml down`.

> Порт совпадает с prod (18082): не запускайте prod- и dev-контейнеры одновременно.

## Сценарий

Страница стартует в светлой теме. Первый клик по камню включает физику: камень падает вниз, а страница скроллится к нижней зоне. После падения включается тёмная тема.

Панель параметров открывается кнопкой в правом верхнем углу. В ней регулируются масса камня, тяготение и сила руки. От этих значений зависит время удержания и высота подъёма. Победа фиксируется, когда камень помещён в верхнюю цель.
