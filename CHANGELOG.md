# Changelog

## Unreleased

### Добавлено

- Совместные realtime-сессии по ссылке с одним серверным состоянием камня.
- Последовательное управление несколькими участниками, reconnect и общая физика.
- Docker-деплой Node.js, CI и автоматические unit/integration/smoke-проверки.
- Общие `grab/grabbing`-курсоры участников, автоматические уникальные комнаты и кнопка копирования URL с 400-мс галочкой.
- Удаление пустой комнаты после 10-секундного grace-периода с отменой при reload/reconnect.
- Диапазон массы камня увеличен с 10 до 100.
- Диапазон тяготения расширен до `1–10`.
- Локальная настройка неограниченной длины следа с сохранением выбора в браузере.
- Локальные настройки ливня в панели параметров: сила, кривые и время появления/исчезновения.
- Атомарное сохранение комнат в Docker volume с восстановлением того же `sessionId` и состояния после рестарта контейнера.
- Одноразовый `session.start`, который по первому scroll создаёт общий отпечаток и запускает первое падение без захвата камня.

### Изменено

- Статический nginx-контейнер заменён одним Node.js-сервисом для статики, API и WebSocket.
- Монолитный `index.html` перенесён на React-компоненты, hooks и чистые frontend-модули с Vite-сборкой.
- Dev-контейнер использует Vite HMR и WebSocket/API proxy, production-образ раздаёт hashed `dist`.
- Dev-watcher использует polling Docker Desktop: React/CSS обновляются через Fast Refresh, shared physics перезагружает страницу, а Express перезапускается без recreate контейнера.
- Первый scroll оставляет общий полупрозрачный отпечаток камня; возвращение в него после падения включает светлую тему и ливень без постоянного примагничивания.
- Возврат удерживаемого камня в отпечаток включает настраиваемый ливень вместе со светлой темой; после выхода из отпечатка дождь плавно скрывается по локальным параметрам.
- Высота DOM-сцены увеличена до `10000vh`; камень падает до фактического низа длинной сцены, а старт остаётся в первом экране.
- Сцена переименована в «ЦАРИ ДОЖДЯ», начинает с тёмной темы и доступной вертикальной прокрутки; камень и `cursor-grab*` включаются только после достижения низа.
- След переведён на viewport-canvas и скрыт по умолчанию; checkbox «Показывать след» включает видимость без сброса накопленных точек.
- Кнопка сессии удалена из панели параметров; копирование осталось только в верхней кнопке.

### Исправлено

- Fast Refresh полностью очищает старый runtime: снимает DOM listeners, отменяет timers/fetch и не запускает повторный WebSocket reconnect.
- Если комната действительно истекла или store повреждён, клиент автоматически создаёт новую рабочую комнату и обновляет URL.
- Вход без ID комнаты выполняет полноценный переход на `/?session=<id>`, поэтому новая общая сцена принимает одноразовый старт по первому scroll.
- Reload последней вкладки сохраняет фазу, положение, скорость, отпечаток, след и общую физику комнаты.
- Изменения общей физики, сделанные во время reconnect, отправляются после восстановления WebSocket и не перетираются промежуточным snapshot.
- Камень больше не уменьшается после отделения от руки.
- Повторяющиеся realtime-снимки больше не перезапускают таймер скрытия ливня.
