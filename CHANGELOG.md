# Changelog

## Unreleased

### Добавлено

- Совместные realtime-сессии по ссылке с одним серверным состоянием камня.
- Последовательное управление несколькими участниками, reconnect и общая физика.
- Docker-деплой Node.js, CI и автоматические unit/integration/smoke-проверки.
- Общие `grab/grabbing`-курсоры участников, автоматические уникальные комнаты и кнопка копирования URL с 400-мс галочкой.
- Немедленное удаление комнаты после закрытия вкладки последним участником.
- Диапазон массы камня увеличен с 10 до 100.
- Локальная настройка неограниченной длины следа с сохранением выбора в браузере.

### Изменено

- Статический nginx-контейнер заменён одним Node.js-сервисом для статики, API и WebSocket.
- Dev-контейнер отслеживает bind-mount изменения polling-наблюдателем и автоматически обновляет открытую страницу.
- Первое касание оставляет общий полупрозрачный отпечаток камня; возвращение в него после падения автоматически останавливает и примагничивает камень.
- Высота общей сцены увеличена с 500vh до 10000vh с сохранением масштаба серверной физики.

### Исправлено

- Верхняя кнопка копирования ссылки остаётся доступной после перезагрузки, во время переподключения и у истёкшей сессии.
- Вход без ID комнаты выполняет полноценный переход на `/?session=<id>`, поэтому новая общая сцена сразу принимает первый захват камня.
